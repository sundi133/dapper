# Resources
{: .no_toc }

Everything you need to evaluate Dapper before adopting it and to operate it day to day: real assessment reports from public vulnerable apps, an honest account of where your code and data go, a problem-oriented troubleshooting guide, the questions teams ask before rolling it out, and the scope and legal guidance that governs responsible use.

If you are still deciding whether Dapper fits your team, start with [Sample reports]({{ '/resources/sample-reports' | relative_url }}) to see what a run actually produces, then read the [FAQ]({{ '/resources/faq' | relative_url }}) and [Disclaimers]({{ '/resources/disclaimers' | relative_url }}). If you are already running it and something is wrong, jump straight to [Troubleshooting]({{ '/resources/troubleshooting' | relative_url }}).
